参考资料目录
================

用途
----
把笔记、FAQ、排障记录、截图说明、导出的文档等放在 documents/ 子目录里。
后续在微信助手或本地检索遇到问题时，可优先参考这里的资料。

目录结构
--------
reference/
  README.txt          本说明（可提交 git）
  documents/          你的资料放这里（默认不提交 git）

建议放什么
----------
- BLE / SDK 常见问题与解决办法
- 项目配置说明、引脚表、协议笔记
- 踩坑记录（现象 + 原因 + 处理）
- 从企微/邮件导出的技术说明 .txt / .md

命名建议
--------
用能搜到的关键词命名，例如：
  ble_central_连接失败.md
  pwr_浅睡唤醒配置.txt
  ota_crc_问题记录.md

编码
----
文本文件建议保存为 GB18030（与项目一致），UTF-8 也可自动识别。

与助手的关系
------------
- Cursor AI 正常时会结合整个 ble52 仓库回答
- Bridge 不可用时，local_search 会检索 documents/ 里的 .txt/.md 等
- 也可直接打开本目录查阅

示例：在 documents/ 新建 my_notes.md，写入问题与结论即可。
